前言：2026年，AI短剧已经不是"能不能做"，而是"怎么做才快"#

打开抖音，你可能已经刷到过AI生成的短剧——霸道总裁、赘婿逆袭、古风虐恋。半年前大家还在争论"AI短剧能不能看"，现在已经有团队做到日更一条，单月产出30条以上。

这不是大厂的专利。我认识的几个个体创作者，一台电脑、¥100/月的API预算，就搭出了一条从剧本到成片的完整流水线。他们不谈艺术，谈的是"制片工业化"——标准化节点、可复用的资产库、模型分工协作。

这篇文章，我会把这条流水线拆成五个步骤，每一步都给你可以直接套用的框架和模板。它不是让你做出奥斯卡短片，而是让你用最低成本、最快速度，稳定产出能过审、能吸量的AI短剧。

第一步：AI编剧——不是让AI瞎写，而是你当主编，AI当枪手#

很多新手的第一反应是：让ChatGPT写一个完整的剧本。然后拿到手一看——情节平淡、对话冗长、毫无网感。这不是AI不行，是你的编剧指令系统没搭起来。

1.1 爽点节奏：每30秒一个钩子#

短剧不是电影。用户在划屏幕，你的竞争对手是下一个视频的拇指滑动。爽点密度决定了完播率。

一个可复用的公式：

每集 = 1个冲突引爆点 + 1次反转/打脸 + 1个悬念尾钩

实操指令模板（直接丢给Claude/GPT）：

【角色设定】你是一位精通抖音短剧节奏的编剧
【任务】生成一集时长90-120秒的短剧剧本
【类型】都市赘婿逆袭
【节奏要求】
- 前10秒必须出现冲突事件（打脸/羞辱/危机）
- 第30-50秒出现第一次反转
- 第70-90秒出现打脸高潮
- 结尾3秒内抛出一个钩子（悬念句/动作中断）
【格式要求】
- 用"场号 场景 日/夜"标记每一场
- 对话不超过3句即切换
- 每场标注"信息释放点"（观众在这里知道了什么）

1.2 人物公式：让AI记住角色，而不是每集重新发明#

AI编剧最大的坑是人物漂移——第一集主角沉稳内敛，第三集突然变成话痨逗比。

解法：在每次生成剧本前，把角色卡片贴在Prompt最前面。

【角色卡-男主-林泽】
- 身份：豪门弃少，隐姓埋名做外卖员
- 性格：表面卑微隐忍，内核果断狠辣
- 口头禅："不着急"
- 行为标签：永远不先动手；反击前必有一个摘手套/挽袖口的动作
- 情绪禁区：不在人前哭；愤怒时反而微笑

这套东西不是一次写完的。每写一集，你会发现角色有了新的行为模式，就回写到角色卡里。十集过后，角色卡就是你的核心IP资产。

1.3 审核红线：避开这五个雷区，过审率提升90%#

短剧平台的审核比你想象的敏感。以下是2026年短剧审核的高频驳回原因：

雷区 具体表现 避坑方法
价值观导向 拜金、以暴制暴被美化 反派作恶必须有代价，主角复仇走法律途径
封建迷信 鬼魂、转世、算命应验 改成"梦境"“幻觉"“心理暗示”
涉政隐喻 官场斗争、警察负面形象 公职人员一律正面或中性
低俗擦边 过度裸露、性暗示镜头 AI生成时加 negative prompt: “nsfw, bikini, lingerie”
侵权风险 角色撞脸明星、BGM未授权 生成后做人脸比对，BGM用平台曲库

关键认知：审核不是在成片后才考虑的。它在剧本阶段就应该被过滤掉。改一句台词比改一段视频便宜一万倍。

第二步：角色与场景资产——有了"资产图”，你才不是每次重开#

这一步是区分"玩票"和"制片"的分水岭。

2.1 角色资产图：6级角色模板#

抖音博主 @AI图图 提出的6级角色模板是目前最实用的体系，核心逻辑是：从粗到细逐级锁定角色特征，而不是一步到位。

级别 锁定内容 工具 产出物
L1 性别/年龄段/体型轮廓 Midjourney / SD 角色剪影参考图
L2 发型/脸型/五官比例 Seedance 角色参考 面部特征锁定
L3 服装/配饰/颜色体系 图生图保持一致性 全身定装照
L4 表情包（喜怒哀乐惊） 表情驱动+ControlNet 表情参考集
L5 动作特征（走路/坐姿/手势） 视频生成测试 动作风格卡
L6 角色在不同场景中的统一性 多场景一致性测试 角色跨场景验证

实操要点：L1-L3 是一次性投入，L4-L6 是持续维护。前期别追求L6完美——先把L3做扎实，角色在剧里的服装颜色+发型保持一致，观众就能认出来。

2.2 场景机位编号：给每个场景一个"身份证"#

别用"客厅"“办公室"这种模糊描述。给每个主场景一个编号和固定机位：

场景编号：S01-林家老宅-客厅
┌─────────────────────────┐
│  CAM-A  全景  │  沙发区  │
│  CAM-B  中景  │  茶几对话│
│  CAM-C  特写  │  门口    │
│  CAM-D  过肩  │  窗户方向│
└─────────────────────────┘

场景编号：S02-周氏集团-总裁办公室
...

好处：

  • 告诉AI生成工具"用S01的CAM-B角度”，角色比例和构图一致性大幅提升
  • 剪辑时不需要苦恼"这段切什么角度"——机位编号就是你的分镜选项
  • 多人协作时，所有人都知道"这场戏用S01-CAM-B"是什么画面

第三步：分镜脚本——Shot Task Card，让每一帧都有"导演意图"#

普通分镜只告诉你"拍什么"。Shot Task Card 告诉你**“为什么要这样拍”**。

3.1 Shot Task Card 四要素#

每张卡片包含四个指令层：

【Shot Task Card #012】

导演意图：此刻观众应该感到"压抑→突然希望"
信息释放：窗外出现救援车队（观众知道，主角不知道）
运动变量：镜头从俯拍缓慢抬升到平视，配合主角抬头动作
表演路线：主角从画面右侧蹲姿→站起→向左移动2步→停住凝视窗外

技术参数：
- 时长：4秒
- 景别：中全景 → 中景（镜头抬升过程中自然变化）
- 运镜：缓慢上升 + 微弱手持晃动（不安感）
- 光线：逆光剪影 → 面部逐渐被窗光照亮

3.2 卡片到Prompt的翻译#

Shot Task Card 是给人看的，给AI的是翻译后的指令：

Seedance/Kling版本（中文优先）：

一个男人从蹲姿缓慢站起，镜头从俯拍抬升到平视，逆光剪影逐渐转为面部被窗外暖光照亮。背景是昏暗的废弃厂房。手持镜头轻微晃动。电影级光影，4K。

LTX版本（英文，简略版用于快速试错）：

A man rises from crouching, camera tilts up from high angle to eye level, backlit silhouette transitions to warm window light on face. Abandoned factory. Slight handheld shake. Cinematic lighting.

3.3 批量生产：一集=20-30张Shot Task Card#

一集90秒的短剧大约需要20-30个镜头。新手最容易犯的错误是一张一张地生成——速度慢且前后镜头割裂。

高效做法：用AI一次性生成整集的Shot Task Card序列，人工审核后批量投喂视频生成工具。

第四步：视频生成——三模型分工，每一分钱都花在刀刃上#

这是流水线里最烧钱也最考验决策力的环节。2026年的模型格局已经很清楚：没有银弹，只有分工。

4.1 三模型能力矩阵#

模型 核心优势 致命短板 最佳角色
Seedance 2.0 中文理解顶级、角色一致性最好、运镜精准 单次时长5-7秒、动作幅度受限 主力生成
Kling 2.0 物理模拟真实、大幅动作流畅、最长10秒 中文理解弱于Seedance、一致性略逊 动作戏、打斗戏
LTX Video 开源免费、生成极快（2秒/条）、本地运行 画质上限低、细节粗糙 试错预演层

4.2 两层协作：LTX预演 → Seedance/Kling定稿#

这是整个流程中最能省钱的一个策略：

┌─────────────────────────────────────┐
│ Shot Task Card #012（4个变体prompt）│
└──────────────┬──────────────────────┘
               ↓
     ┌─────────────────┐
     │  LTX 试错层      │  ← 免费，2秒生成4条
     │  快速预演4条     │
     └────────┬────────┘
              ↓ 人工挑出最优1条
     ┌─────────────────┐
     │  Seedance 定稿层 │  ← ¥0.6/条，只生成精品
     │  高画质最终渲染   │
     └─────────────────┘

逻辑：LTX的质量不够上架，但它能让你零成本试错——构图对不对、角色位置合不合理、运镜方向有没有反。肉眼过一遍，把明显翻车的淘汰掉，只把验证通过的变体送到Seedance/Kling做最终渲染。

一条Seedance 1080p成本约¥0.6/秒，一集90秒理论成本¥54。用LTX筛掉50%的无效生成，实际成本可以压到¥30以内。

4.3 模型分工实战：什么场景丢给谁#

镜头类型 推荐模型 原因
对话、情感特写 Seedance 面部稳定、情绪细腻
打斗、追逐、大幅动作 Kling 物理模拟强，动作不生硬
场景空镜、过渡镜头 LTX 画质要求低，免费不心疼
关键反转/高潮镜头 Seedance（生成3选1） 容错预算，确保名场面不崩
镜头连贯性测试 LTX批量预演 零成本验证分镜逻辑

第五步：后期剪辑合成——从"一堆片段"到"一条短剧"#

5.1 工具链选择#

环节 工具 理由
剪辑 剪映专业版 / Premiere 剪映对竖屏短剧优化极好，字幕模板丰富
配音/TTS Edge-TTS（免费）/ 火山引擎TTS 多角色音色分离，情绪可调
字幕 剪映自动识别 + 样式套用 短剧字幕是视觉元素，不是纯信息
BGM/SFX 剪映音效库 + YouTube Audio Library 注意版权，商用需确认

5.2 MoneyPrinterTurbo 思路借鉴#

开源项目 MoneyPrinterTurbo 的架构对短剧流水线很有参考价值：

  • TTS多Provider切换：主用Edge-TTS（免费稳定），备选Azure/火山引擎。一个挂了自动切下一个。
  • 字幕渲染管线：不是简单叠加SRT——字幕位置随画面构图动态调整，确保不遮挡关键信息。
  • 任务队列：素材生成和后期处理解耦，生成完一批立刻开始剪，不需要等全部素材到位。

5.3 短剧后期的三个致命细节#

  1. 转场别花哨。AI生成画面本身就有"不稳定感"，再加炫酷转场=廉价感翻倍。硬切为主，叠化用于时间流逝。
  2. 音效先行。短剧的情绪70%靠声音传递。脚步、开门、衣服摩擦——这些细节音效比BGM重要十倍。
  3. 导出设置：竖屏1080x1920，码率15Mbps以上。平台转码会二次压缩，源片码率不够直接糊成马赛克。

6.2 成本压缩技巧#

  • 批量优惠：Seedance和Kling的API都有"生成失败不扣费"的机制，利用LTX预演把失败率降到最低。
  • 复用资产：角色图和场景图一次生成，全剧30集复用。这部分成本摊薄后近乎为零。
  • GPU择时：LTX本地跑挑电费低谷时段（如夜间）。虽然单次省不了多少，但月产30条的体量下会累积。

新手避坑：十个做了才懂的教训#

  1. 不要追求单条完美，追求流程可重复。第一条短剧可能烂，但你的流水线要保证第三条一定比第一条好。
  2. 先拍"预告片"再拍全片。用3-5个核心镜头做一个15秒预告，验证角色造型和画风方向。方向错了全剧白做。
  3. LTX预演的镜头编号不要乱。文件名规范：EP03_SHOT12_LTX_v2.mp4。否则素材一多直接爆炸。
  4. 不要同时跑超过3条Seedance生成。API有并发限制，超了排队反而更慢。
  5. 角色正脸照存PNG透明底。后续做缩略图、封面、账号头像全部用得到。
  6. BGM先选好再剪。剪辑跟着音乐节奏走，而不是剪完了硬塞一首歌。
  7. 每集字幕样式保持一致。字体/颜色/描边/阴影参数固定成预设，不要每集手动调。
  8. 发布了才算完成。很多创作者做完就放着"再打磨一下"，结果一个月发不出一条。
  9. 平台首发有流量扶持。新账号前5条短剧如果数据好会获得额外曝光。前5条值得多花预算。
  10. 数据复盘比多做更重要。完播率低于30%的集数，回去看是哪个镜头掉的——多半是前10秒冲突不够。

总结：AI短剧的护城河在流程，不在工具#

2026年的AI视频工具每个月都在迭代，今天Seedance强，明天可能被另一个模型碾压。但工具在变，流程不变。

你做出来的第一条AI短剧可能只是60分。但如果你搭好了这条流水线——角色资产库在积累、Shot Task Card模板在迭代、LTX预演+Seedance定稿的协作模式在跑通——第10条就能到75分，第30条能到85分。

月产30条不是"爆肝"的结果。它是标准化流程的结果。

工具会越来越便宜，模型会越来越强。但知道"先做什么、后做什么、什么值得花钱、什么可以免费"的制片思维，才是你真正的壁垒。
