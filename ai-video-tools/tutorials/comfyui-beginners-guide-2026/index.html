前言#

如果你已经开始接触 AI 视频生成，大概率听过一个名字：ComfyUI。

它被很多人称为"AI 图像/视频生成的瑞士军刀"——功能极其强大，但新手第一眼看到那个布满连线和方块的界面时，往往会倒吸一口凉气。

别怕。 这篇文章就是为你准备的。

我会用最友好的方式，带你从零开始上手 ComfyUI。读完你会发现：它没有想象中那么难，而且一旦学会，你将拥有远超传统 WebUI 的创作自由。

📌 预计阅读时间：15 分钟。读完就能跑通第一个工作流。

一、什么是 ComfyUI？#

1.1 一句话解释#

ComfyUI 是一个基于节点的 Stable Diffusion 图形界面。 它把 AI 生图的每个步骤（加载模型、输入提示词、采样、解码图片等）拆成一个个"节点"，你可以像搭乐高一样自由连接它们。

1.2 为什么不用 WebUI？#

你可能会问：Stable Diffusion WebUI（AUTOMATIC1111）不是已经很成熟了吗？为什么还要学 ComfyUI？

对比维度 ComfyUI SD WebUI
学习曲线 较陡（节点式思维） 平缓（传统表单界面）
灵活性 ⭐⭐⭐⭐⭐ 完全自由组合 ⭐⭐⭐ 受限于预置功能
显存效率 ⭐⭐⭐⭐⭐ 极省显存 ⭐⭐⭐ 中等
工作流复用 ⭐⭐⭐⭐⭐ 一键导入/导出 ⭐⭐ 需要手动记参数
视频生成支持 ⭐⭐⭐⭐⭐ 原生支持 ⭐⭐ 需插件
社区生态 爆发式增长 成熟但放缓

简单说：WebUI 像自动挡汽车，ComfyUI 像手动挡——上手慢一点，但一旦熟练，你就能做出别人做不到的事。

1.3 为什么2026年是学 ComfyUI 的最佳时机#

  • 视频生成大爆发：LTX-Video、Wan、Kling、Seedance 等模型都已通过 ComfyUI 节点集成，一个界面搞定所有。
  • 社区空前活跃：ComfyUI Manager 一键安装节点，GitHub 每天都有新工作流分享。
  • 工具链标准化：越来越多的专业创作者把 ComfyUI 作为核心工作流引擎。

二、安装 ComfyUI（三选一）#

2.1 Windows 本地安装（推荐新手）#

Windows 用户有最便捷的一键安装包。

步骤：

  1. 访问 ComfyUI GitHub Releases
  2. 下载最新的 ComfyUI_windows_portable_nvidia.7z（NVIDIA 显卡）或 CPU 版本
  3. 解压到你想安装的目录（建议纯英文路径，例如 D:\ComfyUI）
  4. 双击 run_nvidia_gpu.bat 启动

🔧 注意：首次启动会自动下载依赖，需要5-15分钟，请耐心等待。浏览器会自动打开 http://127.0.0.1:8188。

2.2 Linux 本地安装#

# 克隆仓库
git clone https://github.com/comfyanonymous/ComfyUI.git
cd ComfyUI

# 创建虚拟环境并安装依赖
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 启动
python main.py

💡 WSL用户提示：通过 http://localhost:8188 访问即可，和普通 Linux 体验一致。

2.3 云端运行（免安装、免配置）#

如果你没有好显卡，或者只想快速体验，云端是最佳选择：

三、你的第一个工作流：文生图#

安装完成后，我们来跑通第一个工作流。ComfyUI 启动后默认会加载一个基础文生图工作流。

3.1 界面速览#

打开 http://127.0.0.1:8188，你会看到几个彩色方块被线条连接：

📦 Load Checkpoint (加载模型)
   ↓
📝 CLIP Text Encode (正向提示词) ──→ 🧠 KSampler (采样器) → 🖼️ VAE Decode → 💾 Save Image
📝 CLIP Text Encode (负向提示词) ──┘

这就是一个完整的文生图流水线。别被英文吓到，我来解释每个节点的作用：

节点 作用
Load Checkpoint 加载你的模型文件（比如 SDXL、SD3）
CLIP Text Encode 把你写的提示词转换成 AI 能理解的格式
KSampler 核心：控制怎么"画"——步数、CFG、种子
VAE Decode 把 AI 的内部数据"翻译"成图片
Save Image / Preview Image 保存/预览结果

3.2 下载你的第一个模型#

ComfyUI 不自带模型，需要手动下载。模型放在 ComfyUI/models/checkpoints/ 目录下。

推荐新手模型（SDXL 系列最成熟）：

  • Juggernaut XL（写实风格）：CivitAI 下载
  • DreamShaper XL（艺术风格）：CivitAI 下载
  • SD3 Medium（最新架构）：HuggingFace 下载

🇨🇳 下载加速：CivitAI 和 HuggingFace 在国内下载较慢，可以通过以下方式加速：

  • 使用 hf-mirror.com 镜像站
  • 配置代理下载
  • 使用 AutoDL 等平台的预装镜像（模型已内置）

3.3 运行你的第一张图#

  1. 把下载的模型文件（.safetensors）放入 ComfyUI/models/checkpoints/
  2. 点击界面右侧的 Refresh 按钮
  3. 在 Load Checkpoint 节点下拉选择你的模型
  4. 在正向提示词框输入：a cat wearing sunglasses, sitting on a beach, sunset, 4k, high quality
  5. 在负向提示词框输入：blurry, low quality, distorted
  6. 点击 Queue Prompt（或按 Ctrl+Enter）

🎉 恭喜！ 你的第一张 AI 图就生成了。右边面板会显示生成进度和结果。

3.4 几个关键参数说明#

参数 含义 新手建议
Steps 采样步数 20-30 即可，再多收益递减
CFG 提示词引导强度 7 是比较平衡的值
Seed 随机种子 fixed 保持结果一致，randomize 每次不同
Width/Height 出图尺寸 从 1024×1024 开始

四、进阶：图生视频工作流#

2026 年，ComfyUI 在视频生成领域的生态已经极其丰富。你可以用同一个界面跑 LTX-Video、Wan、Kling 等主流视频模型。

4.1 安装 ComfyUI Manager（必备）#

ComfyUI Manager 是"节点的应用商店"，一键安装各种社区节点。这是你装完 ComfyUI 后第一个要装的东西。

cd ComfyUI/custom_nodes
git clone https://github.com/ltdrdata/ComfyUI-Manager.git

重启 ComfyUI，侧边栏就会出现 Manager 面板。此后安装任何节点，只需在 Manager 里搜索、点击 Install 即可。

4.2 搭建图生视频工作流#

以最流行的开源视频模型 LTX-Video 为例，它速度快、显存要求低，特别适合入门：

通过 ComfyUI Manager 安装以下节点：

  1. 搜索 ComfyUI-LTXVideo → Install
  2. 搜索 ComfyUI-VideoHelperSuite → Install（视频加载/导出工具）

下载 LTX-Video 模型到对应目录：

  • 模型文件 → ComfyUI/models/diffusion_models/
  • VAE 文件 → ComfyUI/models/vae/
  • Text Encoder → ComfyUI/models/clip/
  • （HuggingFace 搜索 Lightricks/LTX-Video 获取所有文件）

工作流思路：

📷 Load Image (你的参考图)
   ↓
📝 CLIP Text Encode (描述你想要的视频)
   ↓
🎬 LTX-Video Sampler → 🎞️ VAE Decode → 💾 Save Video

生成结果是一段几秒钟的视频，可以作为进一步编辑的素材。

4.3 主流视频模型对比（ComfyUI 节点一览）#

模型 特点 显存要求 ComfyUI 节点
LTX-Video 开源、速度快、适合入门 8GB+ ComfyUI-LTXVideo
Wan 2.1 阿里出品、中文理解强、画质好 12GB+ ComfyUI-WanVideoWrapper
Kling 2.0 快手出品、顶级画质（需API） 云端 ComfyUI-KlingAI
Seedance 2.0 字节出品、中文最强（需API） 云端 通过 ARK API 调用
CogVideoX 开源、智谱出品 12GB+ ComfyUI-CogVideoXWrapper
Mochi 1 开源、Genmo 出品、画质优秀 16GB+ ComfyUI-MochiWrapper

五、ComfyUI Manager 和节点生态#

5.1 Manager 能做什么#

ComfyUI Manager 不仅仅是个"应用商店"，它还能：

  • 📦 安装/更新/卸载节点：一键操作
  • 🔍 缺失节点检测：加载别人的工作流时，自动识别缺少哪些节点
  • 🧩 模型下载引导：提示你需要下载哪些模型文件
  • 📸 工作流分享：把你的工作流导出为 PNG（工作流信息嵌入图片）

5.2 推荐必装节点#

节点包 用途
ComfyUI-Manager 节点管理（必装）
ComfyUI-VideoHelperSuite 视频加载/导出
Efficiency Nodes 效率工具合集
ControlNet Aux ControlNet 预处理器
IPAdapter Plus 图片提示/风格迁移
WAS Node Suite 综合工具包（图片处理、文本等）
rgthree 界面增强、快捷节点

5.3 工作流从哪里获取？#

  • OpenArt Workflows：最大的 ComfyUI 工作流分享社区
  • CivitAI：模型 + 工作流
  • ComfyUI Examples：官方示例
  • Reddit r/comfyui：社区讨论

💡 小技巧：下载的工作流图片直接拖进 ComfyUI 界面即可自动加载！

六、常见问题排错#

6.1 “模型放哪了？找不到模型”#

这是新手最常见的困惑。ComfyUI 的模型目录结构如下：

ComfyUI/
└── models/
    ├── checkpoints/        # SD 主模型 (.safetensors/.ckpt)
    ├── vae/                # VAE 模型
    ├── clip/               # CLIP / Text Encoder
    ├── diffusion_models/   # 视频模型的扩散模型
    ├── controlnet/         # ControlNet 模型
    ├── loras/              # LoRA 模型
    ├── upscale_models/     # 放大模型
    └── ...

🎯 记不住？ 在 ComfyUI Manager 中点击 “Model Manager”，可以直观地看到每个目录应该放什么文件。

6.2 “节点显示红色/报错怎么解决？”#

红色节点 = 节点缺失或加载失败。

  1. 打开 ComfyUI Manager → 点击 Install Missing Custom Nodes
  2. 会自动列出缺失的节点，一键安装
  3. 重启 ComfyUI

6.3 “显存不够，怎么办？”#

显存不足（OOM）是最常见的硬伤。解决方案：

方案 效果
使用 GGUF 量化模型 显存降低 50-70%，质量损失极小
启用 –lowvram 模式 启动时加参数，牺牲速度换显存
降低分辨率 从 1024 降到 768 或 512
使用 Tiled VAE 分块处理，避免 VAE 爆显存
云端运行 根本解决方案

💡 GGUF 模型推荐：在 HuggingFace 搜索 gguf sdxl 或 gguf flux，或在 CivitAI 筛选 GGUF 格式。加载 GGUF 需要安装 ComfyUI-GGUF 节点。

6.4 “ComfyUI 启动后自动关闭”#

常见原因：

  • Python 环境问题：检查是否正确安装了依赖 pip install -r requirements.txt
  • 显卡驱动过旧：更新 NVIDIA 驱动到最新版
  • 路径有中文：ComfyUI 目录和模型路径不要包含中文
  • 端口冲突：改启动参数 python main.py --port 8189

6.5 “生成的视频一片黑/花”#

  • 检查 VAE 文件是否正确放置
  • 检查视频模型文件是否完整（对比 HuggingFace 的 SHA256）
  • 某些视频模型需要特定的采样器/调度器组合，查看节点文档
